# Wireframe Overview – Online Appointment System

This section presents the core wireframes designed to visualize the main user journey in the Online Appointment System.  
The goal is to illustrate how users interact with the system, from login to booking and confirmation.

---

##  Purpose
Wireframes demonstrate:
- The logical structure of key screens.  
- The relationship between user actions and system responses.  
- How usability and clarity guide interface design.

They are intentionally minimal — focusing on user flow, not visual styling.

---

##  Included Wireframes

### 1. Login Page (`Login_Page_Wireframe.png`)
**Objective:**  
Allow users (customers or providers) to log in securely.  

**Highlights:**
- Email & Password fields  
- “Forgot Password” link  
- “Login” button  
- Optional “Remember Me” checkbox  

---

### 2. Booking Flow (`Booking_Flow_Wireframe.png`)
**Objective:**  
Let users browse available time slots and make an appointment.  

**Highlights:**
- Provider and Date selector  
- Slot grid (showing available/unavailable times)  
- “Confirm Appointment” button  
- System feedback: “Appointment confirmed”  

---

### 3. Confirmation Page (`Confirmation_Page_Wireframe.png`)
**Objective:**  
Display confirmation details after a successful booking.  

**Highlights:**
- Appointment summary (provider, date, time)  
- “Add to Calendar” and “Cancel Appointment” buttons  
- Notification icon or email confirmation note
